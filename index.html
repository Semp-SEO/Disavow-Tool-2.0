DNS · HTTPS

Aplikacja sprawdza aktywność wpisów z pliku disavow bez żadnego serwera pośredniego — wszystko w przeglądarce. Dla domen (prefiks domain:) używamy DNS over HTTPS. Dla konkretnych podstron (pełny URL) sprawdzamy DNS hosta + dodatkową próbę HTTP.

⚙️ Logika parsowania pliku disavow

Plik może zawierać zarówno blokady całych domen jak i konkretnych podstron. Aplikacja stosuje następującą logikę:

→Jeśli dla danej domeny istnieje wpis domain:example.com oraz podstrona https://example.com/path — podstrona jest ignorowana. Blokada domeny jest nadrzędna.
→Jeśli podstrona istnieje bez blokady domeny — jest analizowana osobno jako wpis URL.
🔍 Domeny — DNS over HTTPS

Zapytanie o rekord typu A. Wyniki:

✓AKTYWNA — NOERROR + rekordy A/CNAME
✗NIEAKTYWNA — NXDOMAIN (domena wygasła)
?NIEOKREŚLONA — timeout lub błąd sieci
🔗 Podstrony — DNS hosta + weryfikacja HTTP

Dla URL-i takich jak https://example.com/path aplikacja wykonuje dwa kroki: sprawdza DNS hosta (example.com), a następnie próbuje fetch w trybie no-cors do pełnego URL. Ze względu na politykę CORS przeglądarek nie możemy odczytać kodu HTTP odpowiedzi — ale możemy stwierdzić czy serwer w ogóle odpowiedział.

✓AKTYWNA (DNS + HTTP) — DNS hosta OK i serwer odpowiedział na żądanie HTTP
~AKTYWNA (tylko DNS) — DNS hosta OK, ale serwer HTTP nie odpowiedział (może blokować, może być down)
✗NIEAKTYWNA — NXDOMAIN dla hosta — domena podstrony nie istnieje w DNS
?NIEOKREŚLONA — timeout lub błąd sieci

Generator buduje gotowy plik disavow.txt zgodny z wytycznymi Google Search Console. Każdy blok jest opcjonalny i niezależny. Uzupełnione bloki automatycznie łączą się w jeden plik.

📋 Format pliku disavow.txt

Plik musi być w kodowaniu UTF-8. Google obsługuje dwa typy wpisów:

# Komentarz — ignorowany przez Google
domain:spammydomain.com
domain:linkfarm.net
https://bad-site.ru/konkretna-strona

Prefiks domain: blokuje całą domenę. Pełny URL blokuje tylko tę konkretną podstronę.
